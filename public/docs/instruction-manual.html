System Overview

The Customer Hub is a unified web application that replaces the separate welcome portal and workflow customizer with a single platform for managing ESM implementations.

The system has two sides:

ESM Staff Dashboard

Portfolio management, document generation, meeting tracking, configuration, and project oversight. Accessed at /dashboard.

Customer Portal

Branded, read-only view of their implementation. Milestones, action items, documents, meetings, and RAID log. Accessed at /hub.

How data flows

The hub does not store project data locally. All data lives in your existing systems and is read on every page load:

Smartsheet → Milestones, Action Items, RAID, Meetings, Documents → Customer Hub
HubSpot → Intake / Onboarding Data → Customer Hub

When you update a milestone in Smartsheet, it appears in the hub on the next page refresh. There is no sync delay or batch process.

Roles

Three ESM staff roles control what you can see and do:

RoleDashboard AccessCapabilities
Admin All projects Everything: user management, global settings, all project configs, document generation, meeting management
SC Assigned projects only Project configuration, document generation, meeting management, customer contact management
PM Assigned projects only View project details, manage meetings, view documents. Cannot modify project configuration.

Logging In

Staff Login

Navigate to the hub URL and click Sign in. Enter your ESM email and password. You'll land on the Staff Dashboard.

Tip
Your session lasts 8 hours. After that, you'll be redirected to the login page automatically. A warning appears 5 minutes before expiry.

If you've forgotten your password, ask an Admin to reset it from the Users page. The system generates a new random password that you'll need to copy and save.

Customer Login

Customers access their portal through a password-based login followed by contact selection:

1

Customer visits the portal URL and enters their project password (set by the SC in project configuration).

2

If the project has multiple contacts, the customer selects their name from the list. This identifies who is accessing the portal.

3

A magic link email is sent to the selected contact. Clicking the link creates a 30-day session.

4

The customer lands on their branded portal with full access to their implementation data.

Important
Only email addresses listed in the project's Contacts section can log in. If a customer can't access the portal, verify their email is added in the project configuration.

Staff Dashboard

Portfolio View

The main dashboard (/dashboard) shows all your assigned projects in a paginated table. Each row displays:

  • Customer name and project name
  • Status — On Track (green), At Risk (amber), Off Track (red)
  • Current phase derived from Smartsheet milestones
  • Go-live date and days remaining
  • SC and PM assignments
  • Sheet status — whether Smartsheet sheets are linked

Click any project row to open the project detail view.

Quick actions from the portfolio

  • Sync HubSpot — Pull the latest intake records from HubSpot and provision any new projects marked "Create Customer Hub = Yes"
  • Send Notifications — Batch send notification emails to customer contacts across projects

Project Detail

The project detail page (/dashboard/[projectId]) is your command center for a single implementation. It has two columns:

Main area (left)

  • Project details card — Go-live date, current phase, products, SC/PM assignments
  • Timeline / Gantt chart — Visual bar chart of all milestones from Smartsheet, color-coded by status, with completion percentages
  • Milestone list — Detailed table of every milestone with dates, status, and percent complete

Sidebar (right)

  • Quick Stats — Days to go-live, milestones completed, integrations done, open action items, RAID items, meetings held, document count
  • Recaps Needed — Amber alert showing meetings marked complete that still need recap emails sent
  • RAID Log summary — Breakdown of Risks, Actions, Issues, and Decisions with open counts
  • Action buttons — Manage Meetings, Documents, Configure Hub, SharePoint Folder, Refresh Metrics, Generate Meetings
  • Portal Activity — When customers last logged in and which pages they viewed
  • Activity Feed — Recent changes across all project data

Changing project status

The status badge at the top of the project detail page is a dropdown. Click it to change between On Track, At Risk, and Off Track. The change is saved immediately.

Exporting a project report

Click the Export Report button to generate a print-friendly summary of the project. This opens your browser's print dialog with a clean layout that hides navigation and buttons.

Documents

The Documents page (/dashboard/[projectId]/documents) manages all generated and uploaded documents for a project.

Generating documents

Click Generate next to any enabled document type. The hub uses Claude AI to assemble the document from your project's Smartsheet data and intake information. Generation typically takes 10–30 seconds. Available types:

DocumentFormatSource Data
Workflow Data TemplateXLSXCustomer-uploaded procurement docs (extracted by AI)
Workflow GuideDOCXSame extracted workflow data, narrated as prose
UAT TrackerXLSXProject scope and milestones from Smartsheet
UAT Completion GuideDOCXProject scope, with testing instructions
Project CharterDOCXHubSpot intake + Smartsheet milestones + project config
Training GuideDOCXModule scope and customer process data
Go-Live ChecklistDOCXPhase data, integration status, UAT results

Once generated, documents are stored as attachments in the Smartsheet Document Repository sheet. Customers can download them from their portal.

Processing customer uploads

When a customer uploads a procurement document (PDF or DOCX) through their portal, the system automatically:

  1. Extracts text from the file
  2. Sends the text to Claude AI for structured workflow extraction
  3. Generates the Workflow Data Template (XLSX) and Workflow Guide (DOCX)
  4. Uploads both to Smartsheet

You can also upload files on behalf of a customer from the staff Documents view.

Document preview

Click the Preview button next to any document to see its metadata: name, type, status, file size, generation date, and download count. For documents with a link URL, an "Open in New Tab" link is provided.

Meetings

The Meetings page (/dashboard/[projectId]/meetings) manages the full meeting lifecycle for a project.

Generating a meeting schedule

Click Generate Meetings on the project detail page. The system creates a weekly meeting schedule from the project start date to the go-live date, writing rows to the Smartsheet Meeting Tracker sheet.

Managing meetings

The meetings list supports two views:

  • List view — Table of all meetings with week, date, status, and action buttons
  • Calendar view — Monthly grid with color-coded dots for each meeting

Use the search bar to filter meetings by week name, date, or status. Filter by "Needs Action" to see meetings that are missing action items or recaps.

Meeting actions

ActionWhat it does
Process TranscriptPaste or drag-drop a meeting transcript (.srt, .vtt, or plain text). Claude AI extracts a structured recap with summary, discussion points, decisions, and action items.
Log Action ItemsManually enter action items from the meeting. Each item gets an owner, due date, and description.
Send RecapEmails the meeting recap to all customer contacts. Includes summary, action items, and notes.
Mark StatusChange the meeting status (Scheduled, Complete, Skipped, Cancelled).
View RecapView the stored recap for a completed meeting.

Configure Hub

The configuration page (/dashboard/[projectId]/config) controls how the customer's portal looks and behaves. Changes are saved with the Save Configuration button at the bottom.

Note
A yellow "Unsaved changes" indicator appears when you've modified the form. If you try to navigate away with unsaved changes, the browser will prompt you to confirm.

Branding

  • Accent color — Sets the color used for nav, buttons, badges, and metric cards in the customer portal. Use the color picker or enter a hex value.
  • Customer logo — Upload a PNG, JPEG, SVG, WebP, or GIF (max 5 MB). The logo appears alongside the ESM logo in the customer portal header. Stored in Smartsheet.

Portal access

  • Customer portal password — The password customers enter to access their portal. Displayed in plain text so you can share it with the customer.

Portal contacts

Add customer contacts who can access the portal. Each contact needs a name, email, and optional role. Only listed contacts can log in via magic link.

  • Add individually — Fill in the name, email, and role fields and click Add
  • Import from CSV — Click Import CSV and select a file with columns: name, email, role (header row optional). Duplicates are skipped.
  • Get Invite Link — Generate a 24-hour direct access link for a specific contact (no password needed)

Hub sections

Toggle which sections are visible in the customer portal. The Dashboard is always shown. Toggle on/off: Intake, RAID Log, Meetings, Documents, Recordings.

Document types

Choose which document types are available for generation. Disabling a type hides it from both the staff Documents page and the customer portal.

Customer links

Add links that appear in the customer portal sidebar. Each link has a label, URL, and icon type (Link, Smartsheet, SharePoint, Document, Video, Calendar, Training).

Smartsheet configuration

Read-only display of linked Smartsheet sheet IDs. Use the Link Sheets button on the project detail page to connect or reconnect sheets.

Administration

Adding a user

1

Click Add User and fill in full name, email, and role (SC, PM, or Admin).

2

Click Create User. A random password is generated and displayed in a green banner.

3

Copy the password immediately — it will not be shown again. Passwords are stored as bcrypt hashes.

Managing users

  • Edit role — Click the role badge in the table to change a user's role via dropdown

Global Settings

Navigate to Settings from the dashboard header. Available settings include email integration configuration and feature flags for the application.

Provisioning New Projects

New projects are created automatically from HubSpot. No manual configuration is required.

Automatic provisioning (recommended)

1

Create the intake record in HubSpot. Fill in the key fields:

HubSpot FieldMaps To
Customer / Institution Legal NameCustomer name in the hub
Implementation Project NameProject name
Modules in ScopeProduct list (semicolon-separated)
ESM Solution ConsultantSC assignment (resolved from HubSpot owner ID)
Target Go-LiveGo-live date
Project TemplateDetermines which Smartsheet template sheets are copied
2

Set "Create Customer Hub" to Yes. This triggers automatic provisioning:

  • Creates the project entry with all HubSpot data
  • Copies Smartsheet template sheets into a new customer folder
  • Seeds the Project Info sheet with customer details and portal password
  • Sets default dates on the Project Plan from the start date
  • Creates a SharePoint folder (if configured)
  • Generates a customer portal password automatically
3

Verify in the dashboard. The project appears in your list. Click into it to confirm Smartsheet sheets are linked and data is populating.

4

Add customer contacts in the project configuration page. Only listed contacts can access the portal.

5

Share the portal URL and password with the customer's project lead.

Manual sync

If the HubSpot webhook didn't fire (network issue, timing, etc.), click the Sync HubSpot button on the dashboard. This finds all intake records marked for hub creation that haven't been provisioned yet and processes them.

Document Generation Deep Dive

The hub generates seven document types using Claude AI. Each type follows the same pattern: collect source data from Smartsheet/HubSpot, send to Claude with a specialized prompt, and format the result into XLSX or DOCX.

How it works

Source Data → Claude AI → XLSX / DOCX → Smartsheet

Workflow extraction (customer uploads)

When a customer uploads a procurement document, the system runs a specialized extraction pipeline:

  1. Text is extracted from the PDF or DOCX file
  2. Claude receives the full text with a system prompt instructing it to identify approval workflows, spending thresholds, approver chains, and business rules
  3. Claude returns structured JSON with up to 15 workflow steps, each containing threshold ranges, approver sequences, and rule details
  4. The JSON is used to populate both the XLSX template (structured data) and the DOCX guide (narrative form)

Tips for best results

  • Ensure Smartsheet sheets have data populated before generating — empty sheets produce empty documents
  • For Project Charter, make sure HubSpot intake data is filled in (customer profile, scope, team contacts)
  • Customer uploads should be clean PDFs or Word docs — scanned images without OCR will not extract properly
  • You can regenerate any document at any time. Each generation creates a new version; previous versions remain in Smartsheet.

Integrations

Smartsheet

The primary data source. Every project links to multiple Smartsheet sheets:

SheetData Provided
Project PlanMilestones, phases, timeline, percent complete
Action ItemsTasks with owner, due date, status, priority
RAID LogRisks, actions, issues, decisions with status and owner
Meeting TrackerWeekly meeting schedule, status, agenda, recap attachments
Integration TrackerIntegration name, status, dates
Document RepositoryGenerated/uploaded document attachments
MetricsKPI values and completion percentages (optional)

Columns are mapped by title, not column ID. If a column is renamed in Smartsheet, the hub will no longer find it. Standard column names are set during provisioning.

HubSpot

Provides customer intake/onboarding data. The hub reads from a custom object type containing 20+ fields covering institutional profile, procurement guidelines, approval workflows, catalog data, and project team contacts.

The integration is read-only — the hub never writes back to HubSpot. Changes to intake data are made in HubSpot and reflected in the hub on the next page load.

Email (Resend)

Handles all transactional email: magic link login emails, meeting recap distribution, question routing from customers, and bulk notification sends.

SharePoint (optional)

If configured, the hub creates customer folders in SharePoint during provisioning and can link to meeting recordings stored in SharePoint. This is optional and the hub works fully without it.

Troubleshooting

ProblemCauseSolution
Customer can't log in Email not in project contacts, or Resend not configured Verify their email is listed in project configuration (case-insensitive match). Check that the Resend API key is set.
Project data isn't updating Normal — data is fetched live Refresh the page. If data still looks wrong, check the Smartsheet sheet directly to confirm the data is there.
Document generation fails Empty Smartsheet sheets or Anthropic API issue Ensure the linked sheets have data. Check that the Anthropic API key is valid and has credits.
Missing project in dashboard Role-based filtering You only see projects where your email matches the SC or PM field. Ask an Admin to verify the assignment, or log in with an Admin account.
Smartsheet sheets not linked Provisioning incomplete or sheets moved Click Link Sheets on the project detail page to reconnect. You'll need the Smartsheet sheet IDs.
"Unsaved changes" won't go away Form state tracking Click Save Configuration to persist changes, or refresh the page to discard.
Meeting recap email not received Resend configuration or spam filter Verify Resend API key is set. Check the customer's spam/junk folder. The from address must be verified in Resend.
Dark mode looks wrong Some elements may not have dark mode styles Toggle dark mode off. If specific elements are unreadable in dark mode, report it for a CSS fix.
